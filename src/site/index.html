Http Builder NG

Http Builder NG is a modern Groovy DSL for making http requests. It requires Java 8 and a modern Groovy. It is built against Groovy 2.4.x, but it doesn't make any assumptions about which version of Groovy you are using. The main goal of Http Builder NG is to allow you to make http requests in a natural and readable way. See the User Guide for more details.

There are two client implementations available:

  • The default "core" implementation based on the core Java `HttpURLConnection' class - you get this implementation by default using the "-core" library.
  • The Apache HttpComponents implementation - provided in the "-apache" library, which depends on the core library.

Artifacts

Warning: As of version 0.11 the project has been split up to allow for cleaner separation of various client implementations. The Apache-based client implementation was moved into a different package and a different project. If you use the Apache-based builder, you will need to modify your code to account for these changes.

Http Builder NG artifacts are available on Bintray, for Gradle you can add one of the the following dependencies to your build.gradle file dependencies closure:

compile 'org.codehaus.groovy.modules:http-builder-ng-core:${project_version}'

or

compile 'org.codehaus.groovy.modules:http-builder-ng-apache:${project_version}'

For Maven, add one of the following to your pom.xml file:

<dependency>
    <groupId>org.codehaus.groovy.modules</groupId>
    <artifactId>http-builder-ng-core</artifactId>
    <version>${project_version}</version>
    <type>pom</type>
</dependency>

or

<dependency>
    <groupId>org.codehaus.groovy.modules</groupId>
    <artifactId>http-builder-ng-apache</artifactId>
    <version>${project_version}</version>
    <type>pom</type>
</dependency>
